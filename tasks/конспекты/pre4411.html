Шаг 2/2

К шагу 1 К вступлению

Вернитесь к первому предложенному варианту решения.

package lab02; import java.util.Scanner; public class Example { public static void main(String[] args) { Scanner scan = new Scanner(System.in); double x; x = scan.nextDouble(); double result; result = x + 7; System.out.printf("%.4f", result); } }

Отдельные части арифметического выражения названы терминами: операторы и операнды. Оператор это действие, а операнд - то над чем оно выполняется. Если отметить их цветом в решаемой задаче, то получится так:

result = x + 7; Строго говоря, "result" это тоже операнд, а "=" это тоже оператор.

В языке поддерживается множество операторов. На данный момент нам интересны следующие:

*умножение /деление +сложение чисел или склеивание строк -вычитание %остаток от деления
Умножение и деление вычисляются раньше сложения и вычитания. Одинаковые операторы вычисляются слева направо.

Операнды намного более разнообразны. Ими могут быть:

  • Литералы. Т.е. числа, написанные в коде.
  • Переменные.
  • Выражения в круглых скобках.
  • Вызовы методов. Такие операторы будут рассмотрены позже.

Выражения в круглых скобках нужны для указания порядка вычисления. Сравните:

//Вариант 1 result = (1 + 2) * 3; //Вариант 2 result = 1 + 2 * 3;

В первом случае результат будет 9, а во втором 7.

Как правило, операнды должны быть совместимого типа. Например "+" можно использовать для работы со строками, а "-", "*" и "/" нельзя. Строки и числа можно склеить в одну строку оператором "+", но оператор "*" в том же выражении приведет к синтаксической ошибке.

Вступление

В предыдущем разборе теории (открыть в новой вкладке) был рассмотрен ввод чисел с клавиатуры.

Сейчас будет разобран пример оформления вычислений в программе.

Прежде чем отдавать компьютеру команду "посчитай результат", нужно понять назначение каждой переменной.

Какие переменные были объявлены в фрагменте программы ниже?

package lab02; import java.util.Scanner; public class Example { public static void main(String[] args) { Scanner scan = new Scanner(System.in); double x; x = scan.nextDouble(); } }

Их всего две:

  • scan - нужна чтобы считывать данные из консоли.
  • x - нужна для хранения значения от пользователя в виде числа (действительного)

Для хранения результата будет полезно создать еще одну переменную. В абстрактной задаче ее можно назвать просто "result". Если же в задаче, к примеру, дата плотность тела и его объем, а нужно найти массу, то переменным нужно дать более осмысленные названия: density, volume, mass.

Добавьте команду объявления переменной для результата и команду для вывода ее в консоль с нужной точностью.
